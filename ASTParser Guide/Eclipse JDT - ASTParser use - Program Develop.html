Eclipse JDT - ASTParser use

Tag: Java, Author: abcweekly Date: 2009-06-03

The last article describes some of the background knowledge as well as how to create a how the ASTParser Here are some examples of AST.

Create ASTParser and Java source file is parsed into AST:

view plainprint?
  1. // Initialize ASTParser  
  2.         ASTParser parser = ASTParser.newParser(AST.JLS3); //initialize    
  3.         parser.setKind(ASTParser.K_COMPILATION_UNIT);     //to parse compilation unit  
  4.         parser.setSource(content.toCharArray());          //content is a string which stores the java source  
  5.         parser.setResolveBindings(true);  
  6.         CompilationUnit result = (CompilationUnit) parser.createAST(null);  

Imports () is called to get a series of import statements in the source file:

view plainprint?
  1. //show import declarations in order  
  2.         List importList = result.imports();    
  3.         System.out.println("import:");  
  4.         for(Object obj : importList) {  
  5.             ImportDeclaration importDec = (ImportDeclaration)obj;  
  6.             System.out.println(importDec.getName());  
  7.         }  

Call the types () method to get the class name:

view plainprint?
  1. //show class name  
  2.         List types = result.types();    
  3.         TypeDeclaration typeDec = (TypeDeclaration) types.get(0);   
  4.         System.out.println("className:"+typeDec.getName());  

The getField method call TypeDeclaration the field defined in class:

A method corresponding to the contents of a block, can be used getBody (); a block can be decomposed into a series of statements, statements () method can be used to obtain:

  5.             while(iter.hasNext())  
  6.             {  
  7.                 //get each statement

ASTParser as part of the programming language compiler, functional mechanisms can be quite powerful. The above examples are just the tip of the iceberg. Using various resolution ASTParser java source files, the user need only specify the source file, call the ASTParser createAST will be able to get the full AST, the rest is according to the actual needs of the AST manipulate various.

Eclipse documentation detailed records of AST types of description, inheritance and so on. In addition, the AST mechanism used in the Eclipse ASTParser have any questions, you can refer to my article: Eclipse JDT - AST entry, which put a lot of relevant information.
